# T4 — Reference examples (one per pattern)

## Why

Teams adopt fastest by copying a working example. For each fetching pattern
we migrate exactly **one** real component and keep it as the canonical
reference. Each example is a team member's guided first migration.

## What

One production reference per pattern:

1. Simple data fetch
2. Fetch driven by user filters
3. Classic paged table
4. Infinite-scroll grid — covered by the T3 pilot
5. Endless list outside grids
6. Create/update/delete with automatic list refresh
7. Page with the inconsistency alert — migrated as a whole by Pavel

A pattern is dropped if no real component of that shape exists.

## Done when

Each pattern has one merged, working example linked from the cookbook (T5).
